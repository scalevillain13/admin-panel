# Админ-панель управления пользователями

Админ-панель в стиле банковских систем: пользователи (CRUD, роли, группы), группы, роли, права доступа (RBAC), лог активности.

## Стек

- **Frontend:** React 18, TypeScript, Vite, Tailwind CSS, React Router, TanStack Query, React Hook Form, Zod, Zustand
- **Backend:** Supabase (PostgreSQL, Auth, RLS)

---

## Как запустить (подробно)

### Шаг 1. Установить зависимости

В папке проекта откройте терминал и выполните:

```bash
npm install
```

Должны установиться все пакеты из `package.json`. Ошибки вида «module not found» после установки обычно решаются повторным запуском `npm install`.

---

### Шаг 2. Создать проект в Supabase

1. Зайдите на [supabase.com](https://supabase.com) и войдите в аккаунт (или зарегистрируйтесь).
2. Нажмите **New project**.
3. Укажите:
   - **Name** — любое имя (например, `admin-panel`).
   - **Database Password** — придумайте и сохраните пароль к БД (он понадобится только для прямого доступа к PostgreSQL).
   - **Region** — выберите ближайший регион.
4. Нажмите **Create new project** и дождитесь создания (1–2 минуты).

---

### Шаг 3. Выполнить миграции (создать таблицы и данные)

1. В левом меню Supabase откройте **SQL Editor**.
2. Нажмите **New query**.
3. Откройте в проекте файл **`supabase/migrations/20250314000001_initial_schema.sql`**, скопируйте весь его текст и вставьте в окно запроса в Supabase.
4. Нажмите **Run** (или Ctrl+Enter). Внизу должно появиться сообщение об успешном выполнении.
5. Создайте ещё один новый запрос, откройте **`supabase/migrations/20250314000002_seed_roles_permissions.sql`**, скопируйте содержимое, вставьте в SQL Editor и снова нажмите **Run**.

После этого в проекте будут таблицы `profiles`, `roles`, `permissions`, `groups`, `activity_logs` и т.д., а также предзаполненные роли (Admin, Operator, Viewer) и права.

---

### Шаг 4. Настроить вход по email

1. В левом меню Supabase откройте **Authentication** → **Providers**.
2. Убедитесь, что провайдер **Email** включён (Enabled).
3. Для быстрого теста без почты можно отключить подтверждение email:
   - В разделе **Email** найдите **Confirm email** и выключите переключатель.
   - Тогда пользователи смогут входить сразу после создания, без перехода по ссылке из письма.

---

### Шаг 5. Создать первого пользователя (админа)

1. Откройте **Authentication** → **Users**.
2. Нажмите **Add user** → **Create new user**.
3. Укажите **Email** и **Password** (минимум 6 символов). Запомните их — это логин и пароль для входа в админ-панель.
4. Нажмите **Create user**.

Пользователь появится в списке. После первого входа в наше приложение для него автоматически создастся запись в таблице `profiles` (это делает триггер в БД).

---

### Шаг 6. Взять ключи API и создать файл `.env`

1. В Supabase откройте **Project Settings** (иконка шестерёнки в левом меню внизу).
2. Перейдите в раздел **API**.
3. Скопируйте:
   - **Project URL** (например, `https://xxxxx.supabase.co`);
   - **anon public** ключ (длинная строка в блоке **Project API keys**).
4. В корне проекта (рядом с `package.json`) создайте файл **`.env`** (если его нет — скопируйте `.env.example` и переименуйте в `.env`).
5. Откройте `.env` и заполните так (подставьте свои значения):

```env
VITE_SUPABASE_URL=https://ваш-проект.supabase.co
VITE_SUPABASE_ANON_KEY=ваш-anon-public-ключ
```

Сохраните файл. Важно: имена переменных должны начинаться с `VITE_`, иначе Vite не подставит их в код.

---

### Шаг 7. Запустить приложение

В терминале в папке проекта выполните:

```bash
npm run dev
```

В консоли появится что-то вроде:

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

Откройте в браузере **http://localhost:5173** (или адрес, который показал Vite). Должна открыться страница входа в админ-панель.

---

### Шаг 8. Войти и назначить себе роль

1. На странице входа введите **Email** и **Password** пользователя, созданного в шаге 5.
2. Нажмите **Войти**. После успешного входа откроется дашборд.
3. Роль по умолчанию у первого пользователя пустая. Чтобы получить полный доступ (роль Admin), назначьте роль вручную в Supabase:
   - Откройте **Table Editor** → таблицу **`profiles`**.
   - Найдите свою строку (по полю `email`).
   - В колонке **`role_id`** откройте выпадающий список и выберите роль **Admin** (или скопируйте ID роли Admin: `a0000000-0000-0000-0000-000000000001` и вставьте в поле).
   - Сохраните строку.

Обновите страницу админ-панели — теперь у вас полные права. Новых пользователей можно создавать уже из интерфейса (раздел «Пользователи») и сразу назначать им роли.

---

### Если что-то не работает

- **«Invalid API key» или ошибки при загрузке данных** — проверьте, что в `.env` указаны именно **Project URL** и **anon public** ключ из Supabase → Project Settings → API, без лишних пробелов и кавычек.
- **После изменений в `.env`** — перезапустите dev-сервер: остановите его (Ctrl+C) и снова выполните `npm run dev`.
- **Страница входа не открывается / вечная загрузка** — откройте консоль браузера (F12 → Console) и посмотрите ошибки; часто это снова неверный URL или ключ Supabase.
- **Таблицы не найдены** — убедитесь, что оба SQL-файла из `supabase/migrations/` выполнены в Supabase SQL Editor без ошибок.

## Деплой на GitHub Pages

Репозиторий уже настроен на деплой через GitHub Actions. Нужно только добавить секреты и включить Pages.

### 1. Секреты репозитория

1. На GitHub откройте репозиторий **admin-panel**.
2. **Settings** → в левом меню **Secrets and variables** → **Actions**.
3. Нажмите **New repository secret** и создайте два секрета:
   - **Name:** `VITE_SUPABASE_URL` → **Secret:** `https://lnhmcbmmhmcsmvplvqld.supabase.co`
   - **Name:** `VITE_SUPABASE_ANON_KEY` → **Secret:** ваш publishable-ключ (тот же, что в `.env`).

### 2. Включить GitHub Pages

1. В том же репозитории: **Settings** → **Pages**.
2. В блоке **Build and deployment**:
   - **Source:** выберите **GitHub Actions**.

Сохранять ничего не нужно — источник уже выбран.

### 3. Запушить код

В папке проекта выполните (подставьте свой логин и репозиторий):

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/admin-panel.git
git push -u origin main
```

После пуша в ветку `main` автоматически запустится workflow **Deploy to GitHub Pages**: сборка и публикация на Pages.

### 4. Где будет сайт

Через 1–2 минуты админка будет доступна по адресу:

**https://ВАШ_ЛОГИН.github.io/admin-panel/**

(замените `ВАШ_ЛОГИН` на ваш GitHub-логин).

Повторный деплой происходит при каждом пуше в `main`.

---

**Деплой на Vercel / Netlify:** подключите репозиторий и в настройках проекта укажите переменные окружения `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.

## Структура

- `src/api/` — запросы к Supabase (users, groups, roles, activity)
- `src/pages/` — страницы: Дашборд, Пользователи, Группы, Роли, Права, Активность
- `src/components/` — DataTable, Modal, Sidebar, Header
- `supabase/migrations/` — SQL-миграции и сид ролей/прав
